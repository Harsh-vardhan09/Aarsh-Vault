#### Function inside function:-
- when we use function inside a function we have to give it type.
- If it has no type we give it `()=>void`
- If it ahs  argument we have to give the return type and argument in it.

```ts
function delayedOutput(fn:()=>void){
    setTimeout(fn,1000);
}

//if it has argument type then in fn:()=>type 
//we have to give type there
delayedOutput(function(){
    console.log("hello");
})
```
*The type of the argument and return is return here its not returning anything so void*

## Interfaces in TypeScript:-

- ***In Above,  we have to repeat the same object datatypes again and again
- **To assign type to the `user` object, we can use object.
- This style of the object makes the code very confusing and hard to understand.
- To deal with this we make type object to use declaration in other function.
- We can also use Interface to input data.

```ts
interface userType{
    firstName:string,
    lastName:string,
    age:number
}
function legal(user:userType){
    if(user.age>=18){
        console.log(true);
    }else{
        console.log(false);
    }
}
```
